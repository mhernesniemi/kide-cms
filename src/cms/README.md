# Liito CMS Core

Code-first CMS built inside Astro. Define collections in `collections.config.ts`, everything else is generated.

## Quick start

```bash
pnpm cms:generate        # generate schema, types, validators, API
npx drizzle-kit generate # create migration SQL from schema diff
npx drizzle-kit migrate  # apply migrations
pnpm dev                 # start dev server
```

## Defining collections

All collections are defined in a single file — `collections.config.ts`:

```ts
import { defineCollection, defineConfig, fields } from "./core/define";

export default defineConfig({
  database: { dialect: "sqlite" },
  locales: {
    default: "en",
    supported: ["en", "fi"],
  },
  collections: [
    defineCollection({
      slug: "posts",
      labels: { singular: "Post", plural: "Posts" },
      timestamps: true,
      drafts: true,
      versions: { max: 20 },
      fields: {
        title: fields.text({ required: true, translatable: true }),
        slug: fields.slug({ from: "title", unique: true }),
        body: fields.richText({ translatable: true }),
        category: fields.select({ options: ["tech", "design"] }),
      },
    }),
  ],
});
```

### Collection options

| Option       | Type                   | Description                                         |
| ------------ | ---------------------- | --------------------------------------------------- |
| `slug`       | `string`               | URL-safe identifier, used as table name and route   |
| `labels`     | `{ singular, plural }` | Display names in the admin UI                       |
| `pathPrefix` | `string`               | Public route prefix (e.g., `"blog"` → `/blog/slug`) |
| `timestamps` | `boolean`              | Auto-manage `_createdAt` and `_updatedAt`           |
| `drafts`     | `boolean`              | Enable draft/published/changed workflow             |
| `versions`   | `{ max: number }`      | Keep versioned snapshots on every save              |
| `auth`       | `boolean`              | Enable password hashing and session auth            |

## Field types

### `fields.text()`

Single-line or multi-line string.

```ts
title: fields.text({ required: true, maxLength: 200 });
```

| Option         | Type      | Description                              |
| -------------- | --------- | ---------------------------------------- |
| `required`     | `boolean` | Fail validation if empty                 |
| `maxLength`    | `number`  | Maximum character count                  |
| `translatable` | `boolean` | Store per-locale in translations table   |
| `defaultValue` | `string`  | Default value for new documents          |
| `indexed`      | `boolean` | Create a database index on this column   |
| `unique`       | `boolean` | Enforce uniqueness at the database level |

**DB column:** `text`
**Admin component:** `<Input />` (single line) or `<Textarea />` (with `admin.rows`)

---

### `fields.slug()`

URL-safe string, auto-generated from another field.

```ts
slug: fields.slug({ from: "title", unique: true });
```

| Option | Type     | Description                                 |
| ------ | -------- | ------------------------------------------- |
| `from` | `string` | Source field name to generate the slug from |

Slugs are automatically generated by converting the source field value to lowercase, replacing spaces and special characters with hyphens.

**DB column:** `text` (unique)
**Admin component:** `<Input />` with auto-sync from source field

---

### `fields.email()`

Validated email address string.

```ts
email: fields.email({ required: true, unique: true });
```

**DB column:** `text`
**Admin component:** `<Input type="email" />`

---

### `fields.number()`

Integer or float value.

```ts
sortOrder: fields.number({ defaultValue: 0 });
```

**DB column:** `integer`
**Admin component:** `<Input type="number" />`

---

### `fields.boolean()`

True/false toggle.

```ts
featured: fields.boolean({ defaultValue: false });
```

**DB column:** `integer` (0/1 in SQLite)
**Admin component:** `<Checkbox />`

---

### `fields.date()`

ISO 8601 date string.

```ts
publishedOn: fields.date();
```

**DB column:** `text`
**Admin component:** `<Input type="date" />`

---

### `fields.select()`

Single value from a predefined list of options.

```ts
category: fields.select({
  options: ["Product", "Design", "Engineering"],
  defaultValue: "Product",
});
```

| Option    | Type       | Description    |
| --------- | ---------- | -------------- |
| `options` | `string[]` | Allowed values |

**DB column:** `text`
**Admin component:** `<Select />`

---

### `fields.richText()`

Block-structured rich text stored as a JSON AST (not HTML, not Markdown).

```ts
body: fields.richText({ translatable: true, admin: { rows: 14 } });
```

The JSON AST format:

```json
{
  "type": "root",
  "children": [
    {
      "type": "paragraph",
      "children": [
        { "type": "text", "value": "Hello " },
        { "type": "text", "value": "world", "bold": true }
      ]
    },
    {
      "type": "heading",
      "level": 2,
      "children": [{ "type": "text", "value": "Subheading" }]
    }
  ]
}
```

Supported node types: `paragraph`, `heading` (level 2-3), `list` (ordered/unordered), `list-item`, `quote`, `text` (with `bold`/`italic` marks).

**DB column:** `text` (JSON serialized)
**Admin component:** Tiptap/ProseMirror editor with formatting toolbar

---

### `fields.image()`

Reference to an uploaded asset. Stores the asset URL or path.

```ts
cover: fields.image({ admin: { placeholder: "/images/default.jpg" } });
```

**DB column:** `text`
**Admin component:** Image picker with upload and media library browser

---

### `fields.relation()`

Foreign key reference to a document in another collection.

```ts
author: fields.relation({ collection: "authors" });
tags: fields.relation({ collection: "tags", hasMany: true });
```

| Option       | Type      | Description                            |
| ------------ | --------- | -------------------------------------- |
| `collection` | `string`  | Target collection slug                 |
| `hasMany`    | `boolean` | Allow multiple selections (JSON array) |

**DB column:** `text` (single ID or JSON array of IDs)
**Admin component:** `<Select />` with document search

---

### `fields.array()`

JSON array of typed items.

```ts
tags: fields.array({ of: fields.text(), defaultValue: [] });
```

| Option | Type          | Description                    |
| ------ | ------------- | ------------------------------ |
| `of`   | `FieldConfig` | Field type for each array item |

**DB column:** `text` (JSON serialized)
**Admin component:** `<Textarea />` (JSON editing)

---

### `fields.json()`

Arbitrary validated JSON object.

```ts
metadata: fields.json({
  defaultValue: {},
  admin: { rows: 6, help: "JSON object for SEO extras." },
});
```

**DB column:** `text` (JSON serialized)
**Admin component:** `<Textarea />` (JSON editing)

---

### `fields.blocks()`

Ordered array of typed block objects. Each block has a `type` and its own fields.

```ts
blocks: fields.blocks({
  translatable: true,
  types: {
    hero: {
      heading: fields.text({ required: true }),
      body: fields.text(),
      ctaLabel: fields.text(),
      ctaHref: fields.text(),
    },
    text: {
      heading: fields.text(),
      content: fields.richText(),
    },
    gallery: {
      images: fields.array({ of: fields.image(), defaultValue: [] }),
    },
  },
});
```

| Option  | Type                                          | Description            |
| ------- | --------------------------------------------- | ---------------------- |
| `types` | `Record<string, Record<string, FieldConfig>>` | Block type definitions |

**DB column:** `text` (JSON serialized)
**Admin component:** `<Textarea />` (JSON editing)

## Common field options

These options are available on all field types:

| Option         | Type                      | Description                                               |
| -------------- | ------------------------- | --------------------------------------------------------- |
| `label`        | `string`                  | Custom label in the admin UI (defaults to humanized name) |
| `description`  | `string`                  | Help text shown below the label                           |
| `required`     | `boolean`                 | Mark as required in validation                            |
| `defaultValue` | varies                    | Default value for new documents                           |
| `indexed`      | `boolean`                 | Create a database index                                   |
| `unique`       | `boolean`                 | Enforce uniqueness constraint                             |
| `translatable` | `boolean`                 | Store in per-locale translations table                    |
| `admin`        | `AdminFieldComponent`     | Admin UI overrides (placeholder, rows, help, component)   |
| `access`       | `{ update?: AccessRule }` | Field-level access control                                |

## Admin UI configuration

Customize list and edit views per collection in `admin/views.ts`:

```ts
export default {
  posts: {
    list: {
      columns: ["title", "category", "_status", "_updatedAt"],
      defaultSort: { field: "_updatedAt", direction: "desc" },
    },
    edit: {
      layout: [
        { fields: ["title", "excerpt", "body"], width: "2/3" },
        { fields: ["slug", "category", "author", "tags", "cover"], width: "1/3" },
      ],
    },
  },
};
```

### List view options

- `columns` — fields to display as table columns
- `defaultSort` — initial sort field and direction

### Edit view options

- `layout` — field groups with width (`"2/3"`, `"1/3"`, `"full"`)

Fields not assigned to a layout group are placed automatically.

## Document status

Collections with `drafts: true` have three visual states:

| Status        | Meaning                             | Dot color |
| ------------- | ----------------------------------- | --------- |
| **Draft**     | Never published                     | Blue      |
| **Published** | Live, no pending changes            | Green     |
| **Changed**   | Published but has newer draft edits | Yellow    |

The "changed" state is derived by comparing `_updatedAt > _publishedAt`. The database stores only `_status` ("draft" or "published") and `_publishedAt` timestamp.

## Hooks

Lifecycle hooks run inside the local API. Define them in `hooks.ts`:

```ts
import { defineHooks } from "./core/define";

export default defineHooks({
  posts: {
    beforeCreate(data, context) {
      // transform data before insert
      return data;
    },
    afterPublish(doc, context) {
      // invalidate cached pages
      context.cache?.invalidate({ tags: ["posts", `post:${doc._id}`] });
    },
  },
});
```

Available hooks: `beforeCreate`, `afterCreate`, `beforeUpdate`, `afterUpdate`, `beforeDelete`, `afterDelete`, `beforePublish`, `afterPublish`.

The `context` object contains: `user`, `operation`, `collection`, `timestamp`, `cache`.

## Access control

Role-based access rules defined in `access.ts`:

```ts
import { defineAccess } from "./core/define";

export default defineAccess({
  posts: {
    read: () => true,
    create: ({ user }) => user?.role === "admin" || user?.role === "editor",
    update: ({ user }) => user?.role === "admin" || user?.role === "editor",
    delete: ({ user }) => user?.role === "admin",
    publish: ({ user }) => user?.role === "admin" || user?.role === "editor",
  },
});
```

Operations: `read`, `create`, `update`, `delete`, `publish`.

## Internationalization

Fields marked `translatable: true` are stored in a separate `_translations` table. Non-translatable fields live on the main table and are shared across all locales.

```ts
// Query default locale (no join)
const post = await cms.posts.findOne({ slug: "hello" });

// Query specific locale (joins translation table)
const post = await cms.posts.findOne({ slug: "hei", locale: "fi" });

// Upsert a translation
await cms.posts.upsertTranslation(postId, "fi", {
  title: "Hei maailma",
  slug: "hei-maailma",
});
```

The admin edit view shows a locale switcher when multiple locales are configured. Only translatable fields are editable per locale; shared fields are shown read-only.

## Local API

All content operations are typed function calls without HTTP overhead:

```ts
import { cms } from "./cms/.generated/api";

const posts = await cms.posts.find({ status: "published", limit: 10 });
const post = await cms.posts.findById(id);
const created = await cms.posts.create({ title: "New post", ... });
await cms.posts.update(id, { title: "Updated" });
await cms.posts.delete(id);
await cms.posts.publish(id);
await cms.posts.unpublish(id);
```

## Generated files

Running `pnpm cms:generate` produces four files in `.generated/`:

| File            | Contents                                 |
| --------------- | ---------------------------------------- |
| `schema.ts`     | Drizzle ORM table definitions            |
| `types.ts`      | TypeScript interfaces per collection     |
| `validators.ts` | Zod schemas for create/update validation |
| `api.ts`        | Typed local API wrappers per collection  |

These files are auto-generated and should not be edited manually.
